执行淘宝自动上架流程。

## 前置检查

1. 确认 Chrome 已以调试模式运行（`启动调试浏览器.bat`，端口 9222）
2. 确认淘宝卖家后台已登录

## 执行方式

### 方式一：图形界面（推荐）

```
cd /d E:\AI工具\淘宝自动上架工具
python app.py
```

- 启动后点「导入文件夹」选择产品目录，再点「开始上架」
- 每次点「开始上架」都会重新上架，同一产品可反复上架
- 界面实时显示步骤进度、日志、成功/失败状态

### 方式二：命令行

```
cd /d E:\AI工具\淘宝自动上架工具
python run_upload.py $ARGUMENTS
```

- `$ARGUMENTS` 为空时，默认扫描：`E:\AI工具\淘宝店运营流\桌布自动生图工作流\产品素材\1号品`
- 也可指定文件夹，例如：`/上架 E:\AI工具\淘宝店运营流\桌布自动生图工作流\产品素材\2号品`

## 监控要点

实时关注以下关键日志：

**主图上传**
- `sucai库已有图片，直接开始` — 库已缓存，速度快
- `sucai库图片已出现（等了Xs）` — 库加载慢，等待了 X 秒
- `sucai库15s内无图片，走本地上传路径` — 首次上传，走文件上传
- `_click_sucai_image[N/3]完成` — 点选重试，最多3次
- `主图 N 上传失败，重开sucai走library路径重试` — 触发caller-level重试

**其他步骤**
- `步骤:X号品|属性` — 属性填写（涤麻、印花图案等下拉选项）
- `步骤:X号品|价格` — SKU 价格库存填写
- `上架成功  商品ID=...` — 成功信号
- `WARNING` / `ERROR` — 需关注的异常

## 完成后汇报

输出汇总表：成功/失败数量、各商品 ID，以及日志文件路径。

## 重要说明

- `dry_run=False`（命令行）/ 图形界面默认真实发布：**会点击发布按钮**，请确认商品信息无误
- 运费模板固定为「光阴童话」（在 `run_upload.py` 顶部 `SHIPPING_TPL` 或设置页修改）
- 日志自动保存至 `E:\AI工具\淘宝自动上架工具\logs\`
- 同一产品可以重复上架，每次执行都从头开始，不受上次结果影响
- 素材库图片缓存后二次上架更快（sucai库已有图片时跳过等待）
